Реализовать:

Нужно реализовать сервис, который принимает список URL изображений, скачивает их и сохраняет файлы на диск. Для каждого изображения надо сохранить метаданные в базу данных.

Что должен уметь сервис:

Сервис получает список ссылок на изображения.
Он скачивает изображения параллельно, но с ограничением на максимальное число одновременных загрузок.
Файлы сохраняются на диск в локальную папку.
Метаданные по каждому файлу сохраняются в базу.
Сервис должен корректно завершаться, если его останавливают, не теряя уже обработанные данные.

Что считается метаданными:

исходный URL
локальный путь к файлу
дата и время начала загрузки
дата и время окончания загрузки
размер файла в байтах
средняя скорость загрузки
статус загрузки: Success / Failed / Cancelled
текст ошибки, если была ошибка

Что должно получиться на выходе:

На диске должна появляться папка с изображениями.
В БД должна быть таблица с результатами загрузок.
В логах должно быть видно:
какой файл начали качать,
какой завершили,
с какой скоростью,
с какой ошибкой, если не получилось.

"https://example.com/image2.png",
"https://example.com/image3.webp"
]
